Jumping the JavaScript Hurdle

Learning HTML and CSS was like skipping through a sunlit meadow.
JavaScript felt like running face-first into a brick wall.

I had a really hard time in my first week of learning JavaScript. I actually struggled so much that I thought, "Am I really cut out for this career? I should just quit. I'm not smart enough." Luckily, I soon discovered that my problems with JavaScript were not the result of a personal failure, but a pedagogical one.

My first introduction to JavaScript was freeCodeCamp's JavaScript Algorithms and Data Structures course, which is still in beta. The very first lesson in that curriculum is a 174-step monster that attempts to teach all of these concepts at once:

  • Variables
  • Data types
  • Arrays
  • Objects (and arrays of objects)
  • Functions
  • Loops
  • Conditionals
  • Query selectors
  • Local and global scope
  • Click events
  • Random number generation
  • Rendering dynamic HTML content via JavaScript

And to top it all off, the system for validating code and providing feedback was broken. If the user makes a syntax error in fCC's code editor, it is unable to tell you what you did wrong or even that you've done anything wrong at all — it simply makes the "Check Your Code" button stop doing anything, which also prevents you from advancing to the next exercise. It's a good thing JavaScript newbies never make syntax errors, isn't it?

Maybe that teaching style works for some people, but it certainly wasn't working for me. I decided to try this silly-looking little site called Scrimba. The difference was night and day. The introductory lesson there is much less ambitious. It takes baby steps and offers a great deal of hand-holding along the way. And you know what? I think that's a smarter approach. Sure, the end result is less impressive, but because it was more achievable, I retained much more of the information.

Here's a metaphor: it's like taking an introductory cooking class. The freeCodeCamp cooking class is a 7-hour workshop that teaches first-time cooks how to make beef wellington in 174 steps. And what if you do any of it wrong? The chef simply crosses his arms and says nothing, while blocking you from trying the next step.

On the other hand, the Scrimba cooking class first teaches you how to make tomato sauce. After that, you learn how to boil noodles. Then the chef asks, "Okay, how do you think we can combine these to make pasta?" Whether you successfully make pasta or not (but you probably will, because it's pretty damn simple), the chef then demonstrates his pasta-making techniques.

I love freeCodeCamp and I'm glad it exists (in fact, I've sung their praises in a previous post), but their new JavaScript curriculum nearly made me quit coding. It's still in beta, so I hope they will revise it. If I hadn't discovered Scrimba, I might written myself off as an utter moron who would never learn programming.

All of this to say: if you aren't learning, try switching teachers.
